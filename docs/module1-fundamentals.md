# Module 1: Modern API Engineering Principles

**Duration:** Day 1, morning (09:30–11:15)
**Location:** `internal/fundamentals/`

## What You'll Learn

- REST vs RPC — when to use each and why
- Idempotency, statelessness, and contract-first design
- Identity and access: AuthN vs AuthZ, policy as code
- Structured logging with Go's `slog` package
- API lifecycle: versioning (v1/v2), graceful sunsetting
- The Agentic Future: designing APIs for AI consumption and Model Context Protocol (MCP)
